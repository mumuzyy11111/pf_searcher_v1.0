## 2026-06-29 人物卡状态记录器第一步

本次新增人物卡状态记录器，作为与“车卡器”“资料查询”并列的第三个主模块。

### 已完成内容

- 新增 `web/status_tracker.html`，提供独立的人物卡状态记录页面。
- 新增 `web/assets/js/status-tracker.js`，支持多角色状态档案、自动保存、JSON 导入导出。
- 新增 `web/assets/css/status-tracker.css`，提供状态记录器页面样式。
- 更新 `web/index.html` 和 `web/assets/js/index.js`，在首页加入“人物卡状态记录器”分组与 `statusTracker` 面板。
- 新增 `tests/test_status_tracker_static.py`，覆盖主入口注册、页面资源、8 个可填写栏位和核心字段。

### 第一阶段范围

本阶段完成“完整可填写”：

- 基础数值：HP、AC、豁免、速度、BAB、攻击加值、CMB/CMD、先攻、种族、属性等。
- 职业能力：职业、等级、每级能力、使用次数、效果和备注。
- 法术：施法职业、法术位、法术 DC、法术列表和备注。
- 当前专长：专长名、类型、来源、效果、常驻状态和备注。
- 当前 Buff：名称、来源、持续时间、剩余时间、启用状态、影响字段和效果。
- 其他生物：召唤物、动物伙伴、魔宠、盟友等的关键状态与效果。
- 奇物：物品名、部位、装备/启用状态、使用次数和效果。
- 剧情备注：当前任务、NPC、地点、线索、战利品、待办和自由备注。

自动计算与联动属于第二阶段，第一阶段只保留结构化字段，为后续计算做准备。

### GitHub 更新带来的新增能力

- `web/conditions.html`：状态查询页面。
- `web/assets/data/conditions.json`：状态规则静态数据。

### 使用说明

默认只生成审计报告：

```powershell
.\Apply-Repair.ps1
```

确认审计结果后再写入修复：

```powershell
.\Apply-Repair.ps1 -Apply
```
