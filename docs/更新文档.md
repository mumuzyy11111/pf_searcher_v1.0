## 2026-07-02 状态记录器改为手动浏览器保存

本次调整人物卡状态记录器的保存策略：页面默认使用临时状态，不再打开时自动读取浏览器旧存档，也不在编辑时自动写入 localStorage。

### 已完成内容

- 每次打开状态记录器默认创建新的临时状态卡。
- 编辑内容只保存在当前页面会话中，刷新或关闭页面会丢失。
- 新增“保存到浏览器”按钮，点击后才将当前状态写入 localStorage。
- 新增“读取浏览器存档”按钮，读取前会在存在未保存改动时要求确认。
- 新增“清除浏览器存档”按钮，只清除 localStorage 存档，不清空当前临时页面内容。
- 保留“导出 JSON / 导入 JSON”作为文件级保存和恢复方式。
- 更新状态记录器资源版本号，避免浏览器继续加载旧保存逻辑。

## 2026-07-02 状态记录器自动数值影响第一版

本次为人物卡状态记录器加入第一版自动计算能力：职业能力、当前专长、当前 Buff、其他生物和奇物可以配置“数值影响”，并在启用后自动计入基础数值。

### 已完成内容

- 在职业能力、当前专长、当前 Buff、其他生物和奇物条目中新增“编辑数值影响”入口。
- 右侧抽屉支持逐条配置数值、影响目标、加值类型、来源、叠加方式和启用状态。
- 基础数值计算会读取已启用的条目影响，并自动应用到 HP、AC、属性、豁免、攻击、CMB/CMD、先攻和速度。
- 基础数值详情栏保留手动来源，并新增自动来源展示，用于追踪哪些 Buff、专长、能力或奇物正在影响该数值。
- 第一版支持固定数值影响；公式型影响、条件触发和完整规则模板后续再扩展。

### 叠加规则

- 负值默认全部叠加。
- `dodge`、`circumstance`、`untyped` 默认叠加。
- 其他同目标、同类型正加值默认只取最高值。
- 每条影响可设置为默认规则、总是叠加、同类型取最高或只展示不参与计算。

- 基础数值：HP、AC、豁免、速度、BAB、攻击加值、CMB/CMD、先攻、种族、属性等。
- 职业能力：职业、等级、每级能力、使用次数、效果和备注。

- 奇物：物品名、部位、装备/启用状态、使用次数和效果。
- 剧情备注：当前任务、NPC、地点、线索、战利品、待办和自由备注。

自动计算与联动属于第二阶段，第一阶段只保留结构化字段，为后续计算做准备。
